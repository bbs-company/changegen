CLI usage

# One-liner — no install needed
npx @bbs-company/changegen

# Since a specific tag, label the release
npx @bbs-company/changegen --since v1.2.0 --version 1.3.0

# Terminal output only (no file write)
npx @bbs-company/changegen --output terminal

# Pipe to a file
npx @bbs-company/changegen --output markdown --no-file > CHANGELOG.md

Try it — hosted API

Send a single POST request and get back a formatted changelog. No git, no local clone.

curl -X POST https://api.changegen.io/api/changelog \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "repoUrl": "https://github.com/bbs-company/changegen",
    "since":   "v0.9.0",
    "version": "1.0.0"
  }'

Response:

{
  "changelog": "## Changelog — 1.0.0  2024-06-15\n\n### Features\n...",
  "stats": {
    "total": 24,
    "summary": "24 commits · 8 features · 5 fixes · 1 breaking"
  }
}

Self-host in 30 seconds

# Clone and build
git clone https://github.com/bbs-company/changegen.git
cd changegen && npm ci && npm run build

# Run with Docker
docker build -t changegen .
docker run -p 3000:3000 \
  -e CHANGEGEN_API_KEY=secret \
  changegen

# Or run directly with Node
CHANGEGEN_API_KEY=secret npm run start:server

Environment variables:
PORT — port to listen on (default: 3000)
CHANGEGEN_API_KEY — Bearer token for API auth (leave unset to disable auth in dev)
